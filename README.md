Fail2Ban Auto Setup: Защита SSH за 60 секунд

    [!NOTE]
    🤖 AI-Generated: Скрипт и документация подготовлены искусственным интеллектом. Это удобно, но в production-среде всегда проверяйте код перед запуском.

Что делает этот скрипт? Автоматически устанавливает и настраивает fail2ban для защиты от брутфорса. Если кто-то введет пароль неправильно 3 раза, его IP отправится в бан на 10 минут. Скрипт фокусируется только на SSH, не затрагивая другие сервисы.
🚀 Быстрая установка

Выполните одну команду от имени root (или через sudo):
Bash

sudo bash -c "$(curl -fsSL https://raw.githubusercontent.com/Zover1337/Fail2Ban-AUTO/refs/heads/main/fail2ban-install.sh)"

Что произойдет во время установки:

    Детекция ОС: Скрипт сам поймет, у вас Debian/Ubuntu или CentOS/RHEL.

    Установка: Скачает пакеты через системный менеджер (apt или dnf/yum).

    Конфигурация: Создаст /etc/fail2ban/jail.local с оптимальными параметрами.

    Backend-фикс: Настроит systemd как источник логов (актуально для новых версий Ubuntu, где нет /var/log/auth.log).

    Запуск: Включит сервис и добавит его в автозагрузку.

📦 Параметры защиты

Скрипт создает файл /etc/fail2ban/jail.local. Вот основные настройки:
Параметр	Значение	Описание
maxretry	3	Количество попыток до блокировки.
findtime	600	Окно времени (10 мин), в течение которого считаются ошибки.
bantime	600	Длительность бана (10 минут).
backend	systemd	Чтение логов напрямую из журнала системы (надёжнее).
🛠 Управление и проверка
1. Проверить статус защиты

Убедитесь, что "тюрьма" (jail) для SSH активна:
Bash

sudo fail2ban-client status sshd

2. Посмотреть забаненные IP

Список всех нарушителей, которые сейчас находятся в бане:
Bash

sudo fail2ban-client get sshd bannedips

3. Разбанить свой IP (если ошиблись)

Если вы сами случайно попали в бан:
Bash

sudo fail2ban-client unban <ваш_IP>

⚙️ Тонкая настройка

Если вы хотите изменить параметры (например, увеличить бан до часа), отредактируйте конфиг:
Bash

sudo nano /etc/fail2ban/jail.local

После правок обязательно перезапустите сервис:
Bash

sudo systemctl restart fail2ban

    [!TIP]
    Нестандартный порт SSH? Если ваш SSH работает не на 22 порту, измените строку port = ssh на ваш номер (например, port = 2222) в секции [sshd].

⚠️ Решение частых проблем
Проблема	Причина	Решение
Ошибка apt update	Бтые репозитории (напр. Ookla)	Удалите лишнее из /etc/apt/sources.list.d/ и запустите скрипт снова.
Jail не найден	Ошибка в синтаксисе	Проверьте конфиг командой sudo fail2ban-client -d.
Логи не видны	Неверный backend	Скрипт использует backend = systemd. Не меняйте это на auto на новых ОС.
