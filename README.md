**Fail2Ban Auto Setup** — это легковесный bash-скрипт, который автоматизирует базовую защиту вашего сервера от перебора паролей (brute-force) по протоколу SSH.

## 🚀 Быстрая установка

Скопируйте и выполните в терминале вашего сервера (от root или через sudo):

```
sudo bash -c "$(curl -fsSL [https://raw.githubusercontent.com/Zover1337/Fail2Ban-AUTO/refs/heads/main/fail2ban-install.sh](https://raw.githubusercontent.com/Zover1337/Fail2Ban-AUTO/refs/heads/main/fail2ban-install.sh))"
```

### Что делает скрипт:
1.  **Определяет ОС:** Работает с пакетными менеджерами `apt` (Debian/Ubuntu) и `dnf/yum` (CentOS/RHEL).
2.  **Устанавливает пакеты:** Загружает актуальную версию `fail2ban`.
3.  **Конфигурирует SSH Jail:** Создаёт файл `/etc/fail2ban/jail.local`, который не перезаписывается при обновлениях системы.
4.  **Решает проблему логов:** Настраивает `backend = systemd`, что критично для современных дистрибутивов, где отсутствует `/var/log/auth.log`.
5.  **Запускает защиту:** Включает сервис и добавляет его в автозагрузку.

| Параметр | Значение | Описание |

## 🛠 Управление и проверка

### Проверка статуса
Чтобы увидеть количество неудачных попыток и текущий список забаненных IP:

```
sudo fail2ban-client status sshd
```

### Разблокировка IP
Если вы случайно заблокировали себя или доверенный адрес:

```
sudo fail2ban-client unban <IP_ADDRESS>
```

### Просмотр логов в реальном времени

```
sudo tail -f /var/log/fail2ban.log
```

## ⚙️ Тонкая настройка

Если вам нужно изменить время бана или количество попыток, отредактируйте конфигурационный файл:

1. Откройте файл:
```
sudo nano /etc/fail2ban/jail.local
```

2. Измените нужные значения в секции `[DEFAULT]` или `[sshd]`.
3. Примените настройки:

```
sudo systemctl restart fail2ban
```

**Совет:** Если ваш сервер использует другой порт (например, 2222), измените значение `port = ssh` на `port = 2222` в секции `[sshd]`.

## ⚠️ Решение проблем

| Проблема | Решение |
| :--- | :--- |
| **Ошибка apt update** | Проверьте `/etc/apt/sources.list.d/` на наличие битых репозиториев и удалите их. |
| **Сервис не запускается** | Проверьте синтаксис конфига: `sudo fail2ban-client -d`. |
| **IP не банится** | Убедитесь, что вы тестируете с другого устройства. |
